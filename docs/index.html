Needed to include to Gemfile:
gem 'ct_table_for', '~> 0.1.15.beta'

Using an ActiveRecord

The two required fields are: a Model and a collection of ActiveRecord, automatically will render all the attributes of the activerecord "

For example by the following way <%= table_for User, @users %>

Using concrete fields of the Model

If the render of specific fields is wanted, the attributes to be shown can be specified.

Passing the options hash as an Array. For example by the following way <%= table_for User, @users options: {attributes: %w(name email tel)} %>

Using helpers

If helpers want to be used in rendering, specific helper's calling methods can be specified. Attributes of the Model have priority in case of having the same name.

Passing the options hash as an Array. For example by the following way <%= table_for User, @users options: {attributes: %w(my_helper_method)} %>

The helper receives as a parameter the instance of the model.

Using concrete fields and actions.

If buttons are wanted for the actions (show, edit, destroy), a hash of actions: {} can be passed to the options hash with the following parameters actions: {buttons: %w(show edit destroy)}. If these actions have a namespace, premodel option has to be passed to actions as a symbol array, by <%= table_for User, @users options: {attributes: %w(name email tel), actions: {buttons: %w(show edit destroy), premodel: [:bo, :admin]}} %>

Actions with text

By default, in the actions the corresponent icon is shown. But if the text is only wanted, the followinf statement must be included in the actions `hash`: icons: false

Additionally, the option icons: true has also to be passed.

<%= table_for User, @users, options: {attributes: %w( id type active confirmed_at name surnames email), actions: {buttons: %w(show edit destroy), icons: false, premodel: [:bo, :admin]}} %>

Custom actions

The system also allows the configuration of customized buttons according to the needs. In order to do it, a pre-established format must be followed: custom|key1:value1|key2:value2|...

The possible values are:

  • icon: key name of the icon to be used, for example eye, trash o gear
  • class: class to add to the icon
  • title: allows to add a floating text (tooltip)
  • link: action link. In this first version only GET actions are allowed over the same table object.

For example:

<%= table_for User, @users, options: {attributes: %w(id type), actions: {buttons: "custom|icon:plus|class:btn-success|link:new_page_bo_admin|title:New page", premodel: [:bo, :admin]}} %>

With sortable fields

If sortered fields are wanted, the parameter :sortable has to be passed along with the attribute. For example: attribute:sortable

It can be done by the following way <%= table_for User, @users, options: {attributes: %w( id:sortable type:sortable name:sortable surnames:sortable), actions: {buttons: %w(show edit), premodel: [:bo, :admin]} } %>

Other options

Images

If the attribute is an image and its size is not specified, is thumb by default. If other size is wanted, it can be passed as a parameter by the following way: avatar:thumb

Nested Model (premodel)

If it is a model which is dependant of another model. For example, we have got a Chapter that belongs to Book, the routes are polymorphic (for example book_chapter_path), the option premodel have to be passed to actions

This option also can be used in routes with namespaces or scopes

premodel must be an array of symbols

For example, <%= table_for User, @users, options: {attributes: %w(position subtitle), actions: {buttons: %w(show edit destroy), premodel: [:bo, :admin, @book, @chapter]} } %>

Attribute with Reference to a Model

If model Chapter which is being represented has an attribute that is a reference to another model Book, the name of Model's attribute can be passed as a referenced attribute, for example book:name or book:isbn and the table will show name/isbn

For example, <%= table_for Chapter, @chapters, options: {attributes: %w(name book:name)} %>

Attribute with Associations to a Model

If model Book which is being represented has an attribute that is an association to another model Chapter, the name of the association chapters can be passed as the attribute and the table will show the number of related elements.

For example, <%= table_for Book, @books, options: {attributes: %w(name chapters )}%>

Attribute with parameter :currency

If model Book which is being represented has an attribute that is a monetary value price, the parameter :currency can be passed along with the attribute and the table will show the price with the symbol of the currency specified in location files

For example, <%= table_for Book, @books, options: {attributes: %w(name price:currency)} %>

If a change of format or symbol is desired, the modification of the corresponding file in the locale is needed.

ca:
  number:
    currency:
      format:
        format: "%n %u"
        unit: "€"

Attribute with parameter :percentage

If model Book which is being represented has an attribute that is a numeric value in percentage discount, the parameter :percentage can be passed along with the attribute and the table will show the price with symbol %

For example, <%= table_for Book, @books, options: {attributes: %w(name discount:percentage)} %>

If changing the precision is needed, the file /config/initializers/ct_table_for.rb must be created/edited and included:

  CtTableFor.setup do |config|
    config.table_for_numeric_percentage_precision = 2
  end

If a change of format is desired, the modification of the corresponding file in the locale is needed.

ca:
  number:
    percentage:
      format:
        format: "%n%"

Attribute with parameter :l

If model Book which is being represented has an attribute genre and its value needs to be located, the parameter :l can be passed along with the attribute and the table will show the located value according to the location files.

For example, <%= table_for Book, @books, options: {attributes: %w(name type:l genre:l)} %>

Translations of the possible values of the corresponding file of locale have to be included:

ca:
  activerecord:
    attributes:
      book:
        type: Tipus
        genre: Gènere
      book/type:
        fiction: Ficció
        nonfiction: No ficció
      book/genre:
        drama: drama
        scifi: Ciència ficció
        action_and_adventure: Acció i aventura
        other: altres

CSS class and id

At options hash, it can be passed:

  • id (String): will apply to the element <table>.
  • class (String): will apply to the element <table>.
  • tr_class (String): will apply to the element <tr>.
  • btn_class (Hash): will apply to the elements <a> of the actions along with the value by default of table_for_default_action_base_class.

For example, <%= table_for User, @users, options: {id: "my-id", class: "my-class", tr_class: "my-tr-class", btn_class: { show: "btn-primary" } %>

<table id="my-id" class="my-class">
  <tbody>
    <tr class="my-tr-class">
    <td><a class="btn btn-primary"...>...</a></td>
    </tr>
  </tbody>
</table>

<style>
  #my-id{ border: 2px dashed red; }
  .my-class{ background: aqua; }
  .my-tr-class{ color: deeppink; }
